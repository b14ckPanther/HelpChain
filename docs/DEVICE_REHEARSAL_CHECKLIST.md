# Helpchain — Device Rehearsal Checklist

Use this checklist for **three full physical-device rehearsals** before presentation.

---

## Pre-demo setup

- [ ] Mac plugged into power
- [ ] All demo devices charged or plugged in
- [ ] Same Wi-Fi or phone hotspot confirmed
- [ ] Firewall allows incoming connections on demo port
- [ ] `npm run build` completed successfully
- [ ] `npm run dev` started — terminal shows **Network Access** URL
- [ ] LAN URL opens on iPhone Safari (`/requester`)
- [ ] Volunteer URL opens on second device
- [ ] Partner URL opens on laptop
- [ ] Director URL open on presenter laptop (not shown to audience)
- [ ] Screen brightness ≥ 80% on all devices
- [ ] Browser zoom 100% (no accidental pinch zoom)
- [ ] Do Not Disturb enabled on phones
- [ ] Routes opened **before** walking on stage
- [ ] Backup: second phone or laptop with volunteer route ready

---

## During rehearsal — verify each step

- [ ] Requester connected pill green
- [ ] Help → Read text → voice/description → seeking
- [ ] Volunteer receives incoming request
- [ ] Accept → matched on both sides
- [ ] Arrival → in progress → volunteer complete → requester confirm
- [ ] Star balance 4 → 5
- [ ] Ratings optional
- [ ] Redeem Haven Café → QR + code visible
- [ ] Partner simulate scan → redeemed
- [ ] Director reset returns baseline on all tabs without manual refresh issues
